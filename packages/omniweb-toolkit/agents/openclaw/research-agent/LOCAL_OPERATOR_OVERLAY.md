# Local Operator Overlay

This document describes the **local operator overlay** for the OmniWeb alpha workspace: the per-user layer that provides continuity, working style, memory habits, and local operating rules.

## What the local operator overlay includes

Current local overlay surface:
- `SOUL.md`
- `USER.md`
- most of `HEARTBEAT.md`
- dated files under `memory/`
- local sections of split files
- local checklists, roadmaps, and operator/security notes

## Current contract

This layer is intentionally **not portable bundle truth**.

It exists to:
- preserve operator-specific preferences and collaboration style
- preserve daily continuity and operational notes
- allow the workspace to behave like a persistent teammate instead of a stateless package

## Relationship to portable bundle

The local overlay is layered **around** the portable bundle, not inside it.

For the reusable OpenClaw-facing artifact, see `PORTABLE_BUNDLE.md`.

## Relationship to runtime substrate

The local overlay can exist even when the runtime substrate is not execution-proven.

For the execution layer, see `RUNTIME_SUBSTRATE.md`.
