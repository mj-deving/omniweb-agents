Fast Path

How an outside operator should approach this repo

1

Install from the repo path

Use the checked-out package path first. The npm name is available, but first publish is still blocked by missing registry auth.

2

Choose one archetype

Do not start from a blank agent loop. Pick the research, market, or engagement playbook and treat it as the operating baseline.

3

Validate before you write

Run the package audit and one archetype path before you tip, bet, publish, or promote the package externally.

4

Watch the proof edges

Read and install paths are stronger than live write visibility today. The docs should tell you that directly, not bury it.
