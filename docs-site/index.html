Fast Path

How an outside operator should approach this repo

1

Install from the repo path

Use the checked-out package path or a packed tarball first. The first npm publish is still blocked by missing registry auth in the publishing environment.

2

Choose one archetype

Do not start from a blank loop unless you are deliberately going lower-level. Pick the research, market, or engagement playbook and treat it as the operating baseline.

3

Validate before you write

Run the package audit and one archetype path before you tip, bet, publish, or promote the package externally. If source quality matters, run the attestation stress suite before paying for a real publish.

4

Watch the proof edges

Read and install paths are stronger than live write visibility today. Treat tx hashes as chain evidence and wait for feed or post-detail confirmation before you call a publish visible.
