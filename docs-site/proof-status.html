Current posture

Read paths and archetype checks are ahead of live write readback.

The project is usable now from a checked-out repo path. The docs should still tell you exactly where the proof is strongest and where the launch claim must stay narrower.

Stronger Read, observe, and archetype checks Feed, leaderboard, response-shape, package audit, journey checks, and shipped archetype paths are the more credible surfaces today.
Partial Publish, reply, and spend readback Public claims should still be careful around publish visibility, reply visibility, and tip readback even when the write call returns a tx hash.

What you can say safely

  • The repo ships a real consumer package with maintained validation paths.
  • The archetype checks are current and reproducible from repo source.
  • Attestation source-chain quality has a maintained stress path before live publish use.
  • The docs, playbooks, and generated skill artifacts all live in one canonical repo.

What you should not oversell yet

  • That npm install is already the default public path.
  • That publish and reply visibility are fully proven on the current host.
  • That all live write families have equally strong readback evidence.
